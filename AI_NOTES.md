# AI Notes

A transparent record of the AI–human collaboration on this project.  
Covers what was generated by the AI assistant (Antigravity / Gemini), what required human judgement, and where AI output had to be corrected.

---

## 🤖 What AI Was Used For

### 1. LLM-Powered Knowledge Extraction (core product feature)
- Wrote the **system prompt** in `worker/groq_service.py` that instructs `llama-3.3-70b-versatile` to extract entities and relationships in JSON mode.
  This includes the worked example (`Elon Musk → SpaceX → 2002 → California`) that steers the model toward contextual relationships, not just subject–verb–object triples.
- Wrote the **`WORKSPACE_NAME_PROMPT`** that generates a concise 2–5 word name for each workspace from the uploaded corpus.

### 2. Backend Scaffolding
- Generated the initial **FastAPI project structure** (`main.py`, `config.py`, `database.py`, `models.py`, `routes.py`), including the `lifespan` async context manager, CORS middleware, and router prefix setup.
- Implemented **JWT Authentication**: Full user signup and login flow leveraging PyJWT and native BCrypt hashing (replacing passlib). Provided full protective routing across the API footprint, alongside forgotten/reset password endpoints.
- Wrote all **Pydantic schemas** (`WorkspaceOut`, `GraphOut`, `NodeOut`, `LinkOut`, `EntityDetailOut`, `SnippetOut`, etc.) with correct field types and `from_attributes = True` for ORM-mode serialisation.
- Wrote all **API route handlers** for workspaces, entities, relationships, health, and stats — including the aggregate-count helper `_build_workspace_out` and the reusable graph builder `_build_graph_out`.
- Implemented the **entity merge algorithm** (reassign relationships, delete self-loops, deduplicate, reassign snippets, then delete the merged entity).
- Wrote the **workspace eviction logic** (`_enforce_workspace_limit`): queries the oldest workspaces and deletes them when the cap is exceeded.

### 3. Async Refactoring
- Refactored all sync route handlers to `async def`.
- Wrapped every blocking LangChain/Groq call inside `loop.run_in_executor(None, partial(...))` so the FastAPI event loop is never blocked during LLM I/O — this was a non-trivial pattern to apply uniformly across `extract_from_text`, `ping_groq`, and `generate_workspace_name`.

### 4. Text Processing
- Wrote the **`chunk_text`** function with sentence-boundary splitting and hard-fallback for sentences that exceed the chunk size limit — important for keeping each LLM prompt within context-window limits.
- Implemented **`normalize_entity_name`**: strips common honorifics and articles (`Mr.`, `Dr.`, `The`, `A`, `An`, `Sir`, etc.) before deduplication so the same person isn't stored as three different nodes.
- Implemented **cross-chunk deduplication** of entities by `(normalized_name, type_lower)`, preferring longer non-article names.

### 5. Frontend — React Components
- Generated all core page components: `Dashboard.jsx`, `GraphView.jsx`, `StatusPage.jsx`, `HelpPage.jsx`.
- Auth UI Additions: Built `Login.jsx`, `Register.jsx`, and `ForgotPassword.jsx` adhering strictly to the cosmic UI design system. Wired them natively into Axios interceptors and global React Router state. Included strict client-side password complexity rules.
- Built the **`Dashboard`**: drag-and-drop zone, file chip list with remove buttons, upload progress tracking, workspace card grid with inline rename, delete, and navigation.
- Built the **AI extraction loading overlay** with animated brain pulse, synapse rings, and a cycling bank of "reasoning" messages that rotate every 3.5 s while the server is processing.
- Built the full **`GraphView`** page: force-directed graph with custom canvas rendering (`paintNode`, `paintLink`), entity sidebar, edit form, merge flow (Shift-click), connect flow (Ctrl-click), add-entity bar, workspace rename, zoom controls, and search/highlight.
- Built the **`StatusPage`**: polls `/api/health` and `/api/stats` and renders live status indicators.
- Built the **`HelpPage`**: full in-app user guide covering all interaction patterns.
- Generated the `api.js` Axios wrapper with one export per backend endpoint.

### 6. CSS Design System
- Generated the entire **`index.css`** design system: CSS custom properties (color tokens, spacing, radius), dark-mode palette with glassmorphism cards, all component styles (buttons, inputs, cards, sidebar, toast, spinner, upload area, graph toolbar, workspace stat badges, AI loader animation keyframes).


### 7. Infrastructure
- Generated `docker-compose.yml` with the three-service setup (Postgres 16, FastAPI, Nginx/Vite).
- Generated both `Dockerfile`s (backend uses `uv` for fast dependency resolution; frontend uses a multi-stage Nginx build).
- Generated `nginx.conf` with a `try_files` rule for React Router's client-side routing and an `/api` proxy pass to the backend.

### 8. Documentation
- Generated the `README.md` with run instructions, project structure, feature tables, and the "not done" backlog.
- Generated this `AI_NOTES.md`.

## 🧠 What Was Designed / Verified Manually

### Extraction Quality
- **Prompt engineering iterations**: the initial prompt produced too many generic `related_to` relationships. Multiple rounds of manual refinement added the rule about Locations and Dates *always* being connected to entities in the same clause, and the worked example that anchors the model's output format.
- **Chunk size tuning**: `CHUNK_SIZE = 2000` was chosen experimentally — too small produced too many partial-context extractions; too large exceeded Groq's token-per-minute quota for free-tier keys.
- **Model selection**: `llama-3.3-70b-versatile` was chosen over smaller models after manually comparing extraction quality on several sample documents.

### Data Integrity
- **FK cascade verification**: manually confirmed that `ON DELETE CASCADE` on `entities → snippets`, `entities → relationships`, and `workspace → *` prevents orphaned rows when a workspace or entity is deleted.
- **Merge edge-case reasoning**: the merge algorithm's step that removes self-referencing edges (`source_id == target_id == keep.id` after reassignment) was identified and added manually — the AI-generated first draft missed this.
- **Duplicate relationship guard after merge**: the deduplication logic that scans all relationships post-merge and keeps the lowest ID was manually specified because the AI output initially only handled the pre-merge duplicate check.

### Database & Connection Pool
- Manually set `pool_size=5`, `max_overflow=10`, and `pool_recycle=300` to match the expected Docker deployment environment (single server, modest concurrency).
- Verified that `pool_pre_ping=True` prevents "server closed the connection" errors after Postgres restarts.

### API Design Decisions
- Decided that the workspace eviction should happen **before** the new workspace is created (not after) so the user's new upload is never immediately evicted.
- Decided the `X-Workspace-Evicted: true` response header should notify the frontend that an old workspace was silently deleted — surfaced as a toast warning in the UI.
- Chose HTTP 409 (Conflict) over 400 (Bad Request) for duplicate entity names / relationships because it is semantically more precise and lets the frontend distinguish the "already exists — jump to it" case from a plain validation error.

### Frontend Interactions
- **Graph physics tuning**: `d3AlphaDecay={0.015}`, `d3VelocityDecay={0.25}`, `charge.strength(-250)`, `link.distance(120)` were iterated manually — the AI-generated defaults produced heavily overlapping nodes on graphs with 20+ entities.
- **Node label placement**: changed from label-below to label-above the node (above the circle, not inside it) after seeing that neighboring edges would cover in-circle text at default zoom.
- **Search dim threshold**: the `alpha = 0.12` value for non-matching nodes was manually tuned so dimmed nodes are still barely visible (giving spatial context) without distracting from the highlighted matches.
- Decided the **"already exists — jump to it"** behaviour in the Add Entity form (pan + zoom to the existing node, open its sidebar) rather than showing a plain 409 error — this was specified manually and the AI implemented it.

### Infrastructure & Environment
- Manually configured the Vite dev-server proxy (`/api → http://localhost:8000`) so that the frontend can be developed locally without CORS issues.
- Verified the Nginx `try_files $uri $uri/ /index.html` rule is needed for React Router — without it, refreshing on `/workspace/3` returns a 404 from Nginx.
- Checked that `GROQ_API_KEY` is only passed as a Docker Compose environment variable and never baked into an image layer.
